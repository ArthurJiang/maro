class maro.rl.algorithms.torch.abs_algorithm.AbsAlgorithm(model_dict: dict, optimizer_opt: Union[dict, tuple], loss_func_dict: dict, hyper_params: object)[source]¶

Bases: abc.ABC

Abstract RL algorithm class.

The class provides uniform policy interfaces such as choose_action and train. We also provide some predefined RL algorithm based on it, such DQN, A2C, etc. User can inherit from it to customize their own algorithms.

Parameters

  • model_dict (dict) – Underlying models for the algorithm (e.g., for A2C, model_dict could be something like {“actor”: …, “critic”: …})

  • optimizer_opt (tuple or dict) – Tuple or dict of tuples of (optimizer_class, optimizer_params) associated with the models in model_dict. If it is a tuple, the optimizer to be instantiated applies to all trainable parameters from model_dict. If it is a dict, the optimizer will be applied to the related model with the same key.

  • loss_func_dict (dict) – Loss function types associated with the models.

  • hyper_params (object) – Algorithm-specific hyper-parameter set.

abstract choose_action(state, epsilon: float = None)[source]¶

This method uses the underlying model(s) to compute an action from a shaped state.

Parameters

  • state – A state object shaped by a StateShaper to conform to the model input format.

  • epsilon (float, optional) – Exploration rate. For greedy value-based algorithms, this being None means using the model output without exploration. For algorithms with inherently stochastic policies such as policy gradient, this is usually ignored. Defaults to None.

Returns

The action to be taken given state. It is usually necessary to use an ActionShaper to convert this to an environment executable action.

The Deep-Q-Networks algorithm.

The model_dict must contain the key eval. Optionally a model corresponding to the key target can be provided. If the key target is absent or model_dict[target] is None, the target model will be a deep copy of the provided eval model.
